## 6. Tech Debt (non-blocking, captured for future hygiene cycle)

### Smoke driver under wall-clock contention (Phase 20 + Phase 21 jointly)

Live `bin/deploy-router.sh check` after a fresh router rebuild surfaces ~14 transient FAILs (Phase 4 SC-P4-C/E, Phase 5 SC-P5-C database failover, Phase 8 envelope codes, Phase 15 MCP-01 Mcp-Session-Id header, Phase 16 RESS-01/02 events, Phase 17 SESS-05 X-Session-ID header). **None are v0.12.0 regressions.** When reproduced individually with `curl` after Ollama has warmed, each contract holds correctly. Root cause: smoke probes hit the router before ollama finishes warming the model on a fresh boot, so first-byte deadlines elapse.

**Mitigation options for a future cycle:**
- Raise per-probe `--max-time` for cold-path smoke gates
- Add an explicit warmup gate at the start of `smoke-test-router.sh` that waits for `/v1/models[*].health.status === 'ok'` on the canonical chat-local alias
- Or: structurally separate "wiring assertions" (HEAD requests, GET /v1/models shape) from "cold-load assertions" (POST chat completions) into distinct sub-suites

Commit `e113192` (`fix(sse): suppress fastify-sse-v2 retry: 3000 preamble breaking OpenAI SDK streaming`) lands on the v0.12.0 chain immediately before Phase 21's first plan.
