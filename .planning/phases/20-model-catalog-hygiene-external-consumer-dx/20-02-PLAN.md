**Part A — router/src/health/backend-probe.ts (NEW):**

Implement the single-backend probe primitive. Exports:

```ts
export type BackendHealthStatus = 'ok' | 'degraded' | 'down' | 'unknown';

export interface BackendHealth {
  status: BackendHealthStatus;
  checked_at: string;  // ISO8601 UTC
}

export interface ProbeResult extends BackendHealth {
  backend: string;
  latency_ms?: number;
}

// Exported so tests + plugin can introspect / override per environment
export const PROBE_ENDPOINTS: Record<string, { method: 'GET'; path: string } | null> = {
  ollama: { method: 'GET', path: '/' },
  llamacpp: { method: 'GET', path: '/health' },
  vllm: { method: 'GET', path: '/health' },
  'vllm-embed': { method: 'GET', path: '/health' },
  'ollama-cloud': null,  // no public /healthz the bearer reaches
};

export async function probeBackend(
  backend: string,
  baseUrl: string,
  opts?: { timeoutMs?: number; fetchImpl?: typeof fetch },
): Promise<ProbeResult>
```

Behavior contract:
- If `PROBE_ENDPOINTS[backend] === null` → return `{backend, status: 'unknown', checked_at: now}` immediately (no network call).
- If `PROBE_ENDPOINTS[backend] === undefined` (unknown backend value) → return `{backend, status: 'unknown', checked_at: now}` with a warn log (defensive — should not happen given LocalBackendEnum, but a future enum extension shouldn't crash the plugin).
- Otherwise: derive `<base>/<path>` from baseUrl (trim trailing /v1 segment from each `backend_url` because health endpoints are root-relative — e.g. http://ollama:11434/v1 → http://ollama:11434/), call fetch with AbortController + `opts.timeoutMs ?? 2000` ms.
  - 2xx → `status: 'ok'`
  - 3xx / 4xx / 5xx → `status: 'degraded'`
  - timeout / fetch reject (ENOTFOUND, ECONNREFUSED, ECONNRESET) → `status: 'down'`
- Always include `latency_ms` for actually-probed backends.
- Never throw — all failure paths return a `ProbeResult` with `status: 'down'` or `'unknown'`.

URL derivation note: probe endpoints are SERVER-root paths (`/`, `/health`), not `/v1/...`. Each model entry's `backend_url` ends in `/v1`. Use `new URL(path, baseUrl.replace(/\/v1\/?$/, '/'))` or strip then concat — explicitly test the strip in Task A2.

**Part B — router/src/plugins/backend-health-plugin.ts (NEW):**

Implement a Fastify plugin (use `fastify-plugin` if the codebase uses it; otherwise plain `async function plugin(app, opts) { ... }` + `app.decorate(...)`):

```ts
export interface BackendHealthDecoration {
  /** Read current cached health for a backend. Returns 'unknown' + now if no entry. */
  get(backend: string): BackendHealth;
  /** Force-refresh the entire set (used by boot probe + lazy 60s expiry path) */
  refreshAll(): Promise<void>;
  /** Used by the route — if any cached entry is older than ttl_sec, refresh ALL before returning. */
  ensureFresh(): Promise<void>;
}

export interface BackendHealthPluginOpts {
  registry: RegistryStore;          // for enumerating distinct backend names
  valkey: Redis | undefined;        // ioredis client; if undefined, plugin works in-memory-only with a warn log
  ttlSec: number;                   // from env.ROUTER_BACKEND_HEALTH_TTL_SEC
  fetchImpl?: typeof fetch;         // injected for tests
  probeTimeoutMs?: number;          // default 2000
}

export const backendHealthPlugin: FastifyPluginAsync<BackendHealthPluginOpts>
```

Behavior contract:
- On `app.ready` (boot): call `refreshAll()` once. Failures fail-open — log warn and continue; in-memory cache holds `{status: 'unknown', checked_at: now}` per backend.
- `refreshAll()`:
  1. Enumerate distinct backend names from `enabledModels(registry.get())` map → set
  2. For each backend: read first matching entry's `backend_url` (use the first enabled entry whose `backend` matches), call `probeBackend(backend, backend_url, {timeoutMs, fetchImpl})`
  3. Run probes in parallel (Promise.all — bounded by N backends ≤ 5)
  4. Write each result to in-memory Map AND `valkey.set('backend-health:' + backend, JSON.stringify(result), 'EX', ttlSec)` if valkey present
  5. Valkey errors are caught and logged (fail-open): in-memory cache still updated
- `get(backend)`:
  - Returns in-memory Map entry if present
  - If not present → `{status: 'unknown', checked_at: now}` (no implicit blocking probe — `ensureFresh` is the documented refresh hook)
- `ensureFresh()`:
  - If any in-memory entry's `checked_at` is older than `ttlSec` seconds → call `refreshAll()`. Otherwise no-op (fast path).
  - Also reads from Valkey first to honor cross-replica freshness — but since this is single-host, in-memory suffices; document Valkey use as defense-in-depth for future horizontal scaling.
- Decorate the app via `app.decorate('backendHealth', { get, refreshAll, ensureFresh })`.
