Create the `scopedIdsPreHandler` sibling middleware that extracts `X-Tenant-ID`, `X-Project-ID`, and `X-Workload-Class` headers; stamp them on `req`; throw 400 on invalid tenant/project IDs; silent-NULL on invalid workload class. Extend the existing `agentIdPreHandler`'s single `req.log = req.log.child({...})` call to include the new IDs — preserving the Pitfall-9 invariant (exactly ONE `req.log = ` in `router/src/`). Register the new preHandler BEFORE `agentIdPreHandler` in `app.ts` so `.child()` sees the stamped fields.

Purpose: POL-03 (workload class extraction) + POL-05 (the "single req.log assignment" half of POL-05 — the request_log column population half is Plan 07). Without this plan, `req.tenantId`/`req.projectId`/`req.workloadClass` are never populated and Plan 07's DB row population is meaningless.

Output: One new middleware file, one minimal patch to agentId.ts, one app.ts patch, two test files (preHandler behaviors + Pitfall-9 grep gate).
</objective>

  import type { FastifyReply, FastifyRequest } from 'fastify';
  import { InvalidAgentIdError } from '../errors/envelope.js';

  declare module 'fastify' {
    interface FastifyRequest {
      agentId?: string;
      _t0?: number;
      // ...
    }
  }

  const AGENT_ID_RE = /^[A-Za-z0-9._:-]{1,128}$/;

  export async function agentIdPreHandler(req, _reply) {
    req._t0 = performance.now();
    const raw = req.headers['x-agent-id'];
    if (raw === undefined) return;
    const value = Array.isArray(raw) ? raw[0] : raw;
    if (typeof value !== 'string' || !AGENT_ID_RE.test(value)) {
      throw new InvalidAgentIdError(typeof value === 'string' ? value : '');
    }
    req.agentId = value;
    req.log = req.log.child({ agent_id: value });  // line 104 — the ONE req.log assignment
  }
