<objective>
Register the `list_models` MCP tool. Unlike the other 4 tools, this one is a read-only registry projection — no backend call, no canonical request, no AbortController. It:
1. Reads the live registry snapshot via `deps.registry.get()`.
2. Filters by `policies.default.model_allowlist` if non-empty (D-10).
3. Projects each entry to `{ id, object: 'model', created, owned_by, capabilities, policy: { cloud_allowed } }` — backend/backend_url/backend_model intentionally omitted (T-3-A2 anti-leak from Phase 3 preserved).
4. Returns the D-03 stamp `"N models available"` + the full projection in structuredContent.

Purpose: Establishes the MCP-side surface for D-10 (filter + annotation). Plan 11 picks up the symmetric HTTP-side change in `/v1/models` + `/v1/models/:id`.

    Body:
    ```ts
    server.registerTool(
      'list_models',
      {
        title: 'List available models',
        description: 'Returns the policy-filtered set of models available via the router, with cloud_allowed annotation per entry. Backend identity is intentionally hidden.',
        inputSchema: { type: 'object', properties: {}, additionalProperties: false },
      },
      async (_args, _extra) => {
        const t0 = performance.now();
        try {
          const reg = deps.registry.get();
          const allow = reg.policies?.default?.model_allowlist ?? [];
          const created = deps.registry.getCreatedAtSec();
          const entries = reg.models.filter((m) => allow.length === 0 || allow.includes(m.name));
          const data = entries.map((m) => ({
            id: m.name,
            object: 'model' as const,
            created,
            owned_by: 'local-llms' as const,
            capabilities: m.capabilities,
            policy: { cloud_allowed: m.policy?.cloud_allowed ?? true },
          }));
          const stamp = `${data.length} models available`;
          deps.metrics.routerMcpToolCallsTotal.inc({ tool: 'list_models', status_class: 'success' });
          deps.metrics.requestDurationSeconds.observe({ protocol: 'mcp', backend: 'none', model: 'none' }, (performance.now() - t0) / 1000);
          return {
            content: [{ type: 'text' as const, text: stamp }],
            structuredContent: { object: 'list', data },
          };
        } catch (err) {
          const e = err instanceof Error ? err : new Error(String(err));
          const env = toOpenAIErrorEnvelope(e);
          const payload = typeof env === 'symbol'
            ? { error: 'client_disconnect', code: 'client_disconnect', message: 'client disconnected' }
            : { error: env.error.type, code: env.error.code, message: env.error.message };
          deps.metrics.routerMcpToolCallsTotal.inc({ tool: 'list_models', status_class: 'server_error' });
          return {
            content: [{ type: 'text' as const, text: payload.message }],
            structuredContent: payload,
            isError: true,
          };
        }
      },
    );
    ```

The duration histogram observes with backend='none' / model='none' to keep prometheus shape consistent.
