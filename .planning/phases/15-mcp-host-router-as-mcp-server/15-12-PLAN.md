    ## MCP Host (Phase 15 — v0.11.0)

    The router exposes its existing capabilities as MCP tools over Streamable HTTP at `POST /mcp`. The endpoint uses the same bearer token as `/v1/*` routes (`Authorization: Bearer ${ROUTER_BEARER_TOKEN}`) — there is no separate MCP credential.

    ### Tools exposed

    | Tool | Wraps | inputSchema source |
    |------|-------|--------------------|
    | `chat_completion` | POST /v1/chat/completions | `ChatCompletionRequestSchema` |
    | `create_response` | POST /v1/responses | `ResponsesRequestSchema` |
    | `create_embedding` | POST /v1/embeddings | `EmbeddingsRequestSchema` |
    | `rerank` | POST /v1/rerank | `RerankRequestSchema` |
    | `list_models` | GET /v1/models | (no inputs) |

    ### Streaming

    Streaming is **NOT supported** inside MCP tool calls in v0.11.0. The `chat_completion` and `create_response` tools silently coerce `stream:true` to `false`. For SSE streaming, use the HTTP routes directly. MCP-native streaming via progress notifications is deferred to a future MCPS-FUT item.

    ### Configuration

    | Env var | Default | Effect |
    |---------|---------|--------|
    | `MCP_ENABLED` | `true` | When `false`, `/mcp` returns 404 |
    | `MCP_SESSION_TTL_SEC` | `3600` | Idle session GC threshold (seconds) |
    | `MCP_GC_INTERVAL_MS` | `1800000` | Session GC sweep cadence (ms) |

    ### n8n MCP Server Trigger integration

    In n8n, add an MCP Server Trigger node pointing at `https://<your-router-host>/mcp` with the bearer token configured under the node's Authorization header. `tools/list` should return the 5 tools; `chat_completion` is the most common invocation.

    ### Observability

    - `router_mcp_active_sessions` (Prometheus gauge) — live count of MCP sessions; useful as a leak canary.
    - `router_mcp_tool_calls_total{tool, status_class}` — counter per tool invocation.
    - Pino log lines on tool calls carry `tool_name`, `mcp_session_id`, plus the existing tenant/project/agent scoped IDs.
    - Each MCP tool call writes one row to the `request_log` table with `protocol='mcp'`.

    ### Scoped IDs

    Scoped IDs (X-Tenant-ID / X-Project-ID / X-Agent-ID / X-Workload-Class) flow from the OUTER `POST /mcp` HTTP request via the existing `scopedIdsPreHandler`. All N tool calls within a JSON-RPC payload — and across the lifetime of an MCP session — share the same scoped identity. To get MCP tool calls in agent-scoped log queries, configure your MCP client (n8n MCP Trigger node, Claude Desktop, etc.) to forward `X-Agent-Id` on the initial `POST /mcp` request.
