<objective>
Register the `chat_completion` MCP tool on the McpServer built by `buildServerForRequest` in Plan 05's plugin. The tool handler:
1. Derives its `inputSchema` from `ChatCompletionRequestSchema` via `z.toJSONSchema()` (D-01 — no hand-authored shape).
2. Runs the canonical pipeline (`applyPreflight` → capability check → build CanonicalRequest with `stream: false` per D-12 → `adapter.chatCompletionsCanonical(canonical, signal)`).
3. Returns the D-02/D-03 dual-shape result on success (assistant text in `content`, full OpenAI response in `structuredContent`).
4. Catches all errors, runs `toOpenAIErrorEnvelope`, and returns `{ ..., isError: true }` (D-04 / MCPS-04).
5. Pushes one `request_log` row via `bufferedWriter` with `protocol: 'mcp'` (D-05) and the outer-request scoped IDs (D-06).
6. Wires `extra.signal` → `AbortController` → adapter `signal` (D-14).

Purpose: Land the most-used MCP tool (chat_completion) — the surface n8n's MCP Trigger node will exercise first. Establishes the canonical tool-handler pattern that 15-07..15-09 plans (responses, embeddings, rerank) mirror.
Output: First registered MCP tool; integration tests still pass (existing /v1/chat/completions HTTP route untouched).
</objective>
