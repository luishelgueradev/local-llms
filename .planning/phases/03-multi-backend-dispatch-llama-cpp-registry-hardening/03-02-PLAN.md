<objective>
Land **Slice B + E** — the `/v1/models` endpoint (OAI-03) and the registry hardening that enforces the 16 GB VRAM envelope at startup (BCKND-04). Together they tighten the contract between operators and the router: drift in `models.yaml` (missing capabilities, over-subscribed VRAM, unknown backend) is rejected loud at boot — not silently at first request.

This plan is a vertical slice: route + schema + types + wiring + tests + the data file (`router/models.yaml`), all moving together. The user can — after this plan — call `GET /v1/models` with the bearer token and see the two-entry catalog from `models.yaml` emitted in the D-C1 shape with a stable `created` timestamp. They can also edit `models.yaml` to declare `vram_budget_gb: 20` for the llamacpp model and observe the router refuse to swap (hot-reload) or refuse to start (cold boot).

Purpose:
- Make `models.yaml` the single source of truth that the runtime **actively enforces** (Phase 2 accepted-but-ignored these fields; Phase 3 reads them).
- Ship the `/v1/models` endpoint Open WebUI (Phase 6) will consume.
- Wire the `factory.ts` from Plan 01 into `app.ts` (this plan owns the app.ts edit; Plan 03 + Plan 04 will add their own bootstrap-time singletons to the same builder).

4. **Rewrite `RegistrySchema`** to use `.superRefine` for the VRAM envelope check. The envelope is read INSIDE the refinement (revision 1; approach b from the original plan) so operators can change `VRAM_ENVELOPE_GB` via `docker compose restart router` without rebundling AND tests can mutate `process.env.VRAM_ENVELOPE_GB` per-test without `vi.resetModules()`:

   ```ts
   export const RegistrySchema = z.object({
     models: z.array(ModelEntrySchema).min(1, 'models.yaml must declare at least one model'),
     backends: BackendsSection,
   }).superRefine((reg, ctx) => {
     // Read env at refinement time (not module load) — allows operators + tests to toggle.
     const envelope = Number(process.env.VRAM_ENVELOPE_GB ?? 16);
     const sums = new Map<string, number>();
     for (const m of reg.models) {
       sums.set(m.backend, (sums.get(m.backend) ?? 0) + m.vram_budget_gb);
     }
     for (const [name, sum] of sums) {
       if (sum > envelope) {
         ctx.addIssue({
           code: 'custom',
           path: ['models'],
           message: `Config error: backend "${name}" declared models sum to ${sum} GB, exceeds VRAM_ENVELOPE_GB=${envelope}. Reduce vram_budget_gb on one or more entries.`,
         });
       }
     }
   });
   ```

5. **Update `Registry` and `ModelEntry` types** — if the file currently exports `type Registry = z.infer<typeof RegistrySchema>` and `type ModelEntry = z.infer<typeof ModelEntrySchema>`, those stay correct (zod's inference picks up the wider `backend` enum and the new `backends?` field automatically). If the types are hand-written elsewhere, update them in lockstep.

6. **Add `createdAtSec` to `RegistryStore` (revision 1 — D-C3):**

   Locate the existing `makeRegistryStore` function (Phase 2). Add an internal `createdAtSec` variable and an accessor:

   ```ts
   export interface RegistryStore {
     get(): Registry;
     getCreatedAtSec(): number;          // NEW — Unix seconds, stable across snapshot lifetime
     _swap(next: Registry): void;
   }

   export function makeRegistryStore(initial: Registry): RegistryStore {
     let snapshot = initial;
     let createdAtSec = Math.floor(Date.now() / 1000);   // NEW — set at boot
     return {
       get: () => snapshot,
       getCreatedAtSec: () => createdAtSec,
       _swap: (next) => {
         snapshot = next;
         createdAtSec = Math.floor(Date.now() / 1000);   // NEW — advances on every successful swap (hot-reload)
       },
     };
   }
   ```

   Watcher already calls `store._swap(next)` only when the new YAML parses cleanly (Phase 2 keep-previous-on-error semantic). That single path is the only place `createdAtSec` advances — failed reloads do NOT touch it. Document this guarantee in a code comment.
