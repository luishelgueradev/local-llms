OpenAI tool_choice variants (request):
- `'auto'` (default with tools) ↔ canonical `{type:'auto'}`
- `'required'` ↔ canonical `{type:'any'}`
- `{type:'function', function:{name:'X'}}` ↔ canonical `{type:'tool', name:'X'}`
- `'none'` ↔ canonical `{type:'none'}` (FINDING 3.4 CORRECTION — D-D3 said "strip tools[]"; 2026 Anthropic supports native {type:'none'})

OpenAI `parallel_tool_calls: false` ↔ Anthropic `tool_choice.disable_parallel_tool_use: true` (modifier on the existing tool_choice object). Allowed on `type: 'auto'`, `type: 'any'`, `type: 'tool'`. For `type: 'none'` the field is meaningless — don't emit.

OpenAI tool message → Anthropic user message tool_result block:
- OpenAI: `{role:'tool', tool_call_id:'call_abc', content:'<string>'}` (single message per tool result)
- Anthropic: `{role:'user', content:[{type:'tool_result', tool_use_id:'call_abc', content:'<string>', is_error?:boolean}, ...]}` — multiple tool_results MUST be in ONE user message's content array (FINDING 3.6).
- is_error round-trip OpenAI → Anthropic: detect when the OpenAI tool content is wrapped as `'{"is_error":true,"result":"..."}'` (JSON-encoded) and lift to canonical `is_error:true`; otherwise default is_error:undefined.
- is_error round-trip Anthropic → OpenAI: when canonical tool_result has `is_error:true`, serialize to OpenAI `content: JSON.stringify({is_error:true, result: tool_result.content})` (FINDING 3.7).

stop_sequences ⇄ stop (ANTHR-08 / D-D5 / Pitfall 6 / FINDING 3.5):
- OpenAI `stop: 'X'` (string) → canonical `stop_sequences: ['X']`.
- OpenAI `stop: ['X','Y']` (array, max 5) → canonical `stop_sequences: ['X','Y']`.
- OpenAI `stop` with > 5 entries → throw ZodError (Anthropic limit; CanonicalRequestSchema has `.max(5)` already from Plan 01).
- Canonical → OpenAI: always emit `stop: string[]` (array form; OpenAI accepts both).
- Anthropic → canonical: pass-through (already arrays).

InvalidImageUrlError + ImageFetchError (NEW typed errors, T-04-01 mitigation per locked D-C4 — added in Plan 04 Task 2 to avoid wave-4 envelope.ts collision with Plan 05; Plan 05's ollama-native-out.ts imports them):
```typescript
export class InvalidImageUrlError extends Error {
  readonly code: 'invalid_image_url' = 'invalid_image_url';
  constructor(public readonly url: string, public readonly reason: 'http_scheme_blocked' | 'private_address_blocked') {
    super(reason === 'http_scheme_blocked'
      ? `Image URL must use https:// scheme; got non-https URL: ${url}`
      : `image URL resolves to a private/loopback address — rejected for SSRF mitigation: ${url}`);
    this.name = 'InvalidImageUrlError';
  }
}

export class ImageFetchError extends Error {
  readonly code: 'image_too_large' | 'image_invalid_content_type' | 'http_error';
  constructor(public readonly url: string, code: 'image_too_large' | 'image_invalid_content_type' | 'http_error', detail: string) {
    super(`failed to fetch image from ${url}: ${detail}`);
    this.name = 'ImageFetchError';
    this.code = code;
  }
}
```
Mapping (all three Image*Error variants):
- `mapToHttpStatus`: `if (err instanceof InvalidImageUrlError || err instanceof ImageFetchError) return 400;`
- `toOpenAIErrorEnvelope`:
  - `InvalidImageUrlError → { error: { message: err.message, type: 'invalid_request_error', code: 'invalid_image_url', param: 'messages[].content[].source.url' } }`.
  - `ImageFetchError → { error: { message: err.message, type: 'invalid_request_error', code: err.code, param: 'messages[].content[].source.url' } }` (the `code` field comes from the per-instance `code` property: `image_too_large`, `image_invalid_content_type`, or `http_error`).
- `toAnthropicErrorEnvelope` inherits via `mapOpenAITypeToAnthropic` (`invalid_request_error → invalid_request_error`).

golden.test.ts runner shape:
```typescript
import { readdirSync, readFileSync, existsSync } from 'node:fs';
import { join } from 'node:path';
import { describe, it, expect } from 'vitest';

const goldenDir = new URL('./golden/', import.meta.url).pathname;
const scenarios = readdirSync(goldenDir).filter(d => /^\d+-/.test(d));

for (const sc of scenarios) {
  describe(`golden round-trip: ${sc}`, () => {
    const dir = join(goldenDir, sc);
    const has = (f: string) => existsSync(join(dir, f));
    const read = (f: string) => JSON.parse(readFileSync(join(dir, f), 'utf8'));

    if (sc === '09-malformed-tool-args') {
      it('openai-in throws InvalidToolArgumentsError on malformed JSON arguments', () => {
        expect(() => openAIRequestToCanonical(read('input-openai.json'))).toThrow(/invalid_tool_arguments|tool_call arguments not valid JSON/);
      });
      return;
    }
    const canonical = read('canonical.json');
    const idOverride = canonical.response?.id;          // deterministic id seam from translator option
    const toolUseIdOverrides = canonical.response?.content
      ?.filter((b: { type: string }) => b.type === 'tool_use')
      .map((b: { id: string }) => b.id);
    if (has('input-openai.json')) {
      it('OpenAI → canonical (request)', () => expect(openAIRequestToCanonical(read('input-openai.json'))).toEqual(canonical.request));
    }
    if (has('input-anthropic.json')) {
      it('Anthropic → canonical (request)', () => expect(anthropicRequestToCanonical(read('input-anthropic.json'))).toEqual(canonical.request));
    }
    if (has('output-openai.json') && canonical.response) {
      it('canonical → OpenAI (response)', () => expect(canonicalToOpenAIResponse(canonical.response, { idOverride })).toEqual(read('output-openai.json')));
    }
    if (has('output-anthropic.json') && canonical.response) {
      it('canonical → Anthropic (response)', () => expect(canonicalToAnthropicResponse(canonical.response, { idOverride })).toEqual(read('output-anthropic.json')));
    }
  });
}
```

NOTE: `canonical.json` is the union `{request, response}` (both halves in one file). The runner reads the `response.id` field as the `idOverride` so the translator produces a deterministic id matching the output fixtures.
