## anthropic-version Sanitization — Implementation Choice

**Where:** `router/src/routes/v1/messages.ts` `sanitizeAnthropicVersion(raw)` helper (called at the top of the route handler, BEFORE the stream:true 501 branch so even the 501 envelope carries the echoed header).

**What it does:**
- If header absent → return `null` → no `reply.header(...)` call → response has NO `anthropic-version` (RESEARCH FINDING 7 — agents test for this; injecting a default would change their detection logic).
- If Fastify gave us a `string[]` (multi-value), pick the first.
- Slice to first 64 chars (defense-in-depth — Node's HTTP parser already caps total request header size).
- Strip `\r` and `\n` (header injection mitigation per T-04-05).

**Why not at request parse:** Node's HTTP parser already rejects raw `\r\n` in header values upstream of any route handler (Fastify returns 400 from a request that didn't even hit our code). The sanitization is therefore **defense in depth**, not the primary line of defense. The integration test ANTHR-05 #7 uses a length-120 plain-ASCII value (no real CR/LF in the request) so the test exercises the slice cap and the regex strip without colliding with Node's parser. A test with a true `\r\n` value would fail in `app.inject()` itself before reaching the handler.

## Centralized Error Handler — req.url Fallback Behavior

**Where:** `router/src/app.ts:setErrorHandler` block (lines ~85–115 after Task 2).

**Behavior matrix:**

| `req.url` value | Envelope path |
|---|---|
| `'/v1/messages'`              | toAnthropicErrorEnvelope |
| `'/v1/messages/'`             | toAnthropicErrorEnvelope |
| `'/v1/messages/count_tokens'` | toAnthropicErrorEnvelope |
| `'/v1/chat/completions'`      | toOpenAIErrorEnvelope    |
| `'/healthz'`                  | toOpenAIErrorEnvelope    |
| `''` or undefined             | toOpenAIErrorEnvelope (fallback) |

The fallback to OpenAI envelope when `req.url` is undefined is deliberate: pre-routing errors (extremely rare; would happen if an `onRequest` hook threw before Fastify populated `req.url`) don't carry an Anthropic-version expectation. The OpenAI envelope is the dominant surface and the safer default — every pre-Phase-4 route already used it. No new failure modes introduced.
