From RESEARCH §"Code Examples — postgres sidecar" lines 630–655 (the LITERAL sidecar shape):
```yaml
pg-backup:
  image: postgres:17-alpine
  container_name: ${COMPOSE_PROJECT_NAME:-local-llms}-pg-backup
  restart: unless-stopped
  networks: [data]
  volumes:
    - ${HOST_DATA_ROOT:-/srv/local-llms}/postgres-backups:/backups
  environment:
    - PGPASSWORD=${POSTGRES_PASSWORD}
  entrypoint: ["/bin/sh", "-c"]
  command:
    - |
      while true; do
        STAMP=$$(date -u +%Y-%m-%dT%H)
        pg_dump -h postgres -U app -d router --format=custom -f /backups/router-$$STAMP.dump || \
          echo "[pg-backup] dump failed at $$STAMP"
        find /backups -name "router-*.dump" -mtime +7 -delete
        sleep 86400
      done
  depends_on:
    postgres:
      condition: service_healthy
      required: false
```

From RESEARCH §"Code Examples — Restore drill script skeleton" lines 776–807 (the LITERAL restore-drill structure):
```bash
#!/usr/bin/env bash
set -uo pipefail
DUMP_FILE="${1:?usage: $0 <dump-filename>}"

# Terminate active sessions
docker compose exec -T postgres psql -U postgres -d postgres -c "
  SELECT pg_terminate_backend(pid) FROM pg_stat_activity
  WHERE datname = 'router' AND pid <> pg_backend_pid();"

# Drop + recreate
docker compose exec -T postgres psql -U postgres -c "DROP DATABASE IF EXISTS router;"
docker compose exec -T postgres psql -U postgres -c "CREATE DATABASE router OWNER app;"
docker compose exec -T postgres psql -U app -d router -c "CREATE EXTENSION IF NOT EXISTS pgcrypto;"

# Restore
docker compose exec -T postgres pg_restore --dbname=router --username=app /backups/"$DUMP_FILE"

# Sanity
ROWS=$(docker compose exec -T postgres psql -U app -d router -tAc "SELECT COUNT(*) FROM request_log")
echo "Restore drill: request_log row count = $ROWS"
[[ "$ROWS" =~ ^[0-9]+$ ]] || { echo "FAIL: count not numeric"; exit 1; }
echo "PASS — restore drill completed without error."
```
NOTE: that example assumes `POSTGRES_USER=postgres` (default superuser). Plan 01 picked Approach A: `POSTGRES_USER=app`. So `docker compose exec -T postgres psql -U postgres` will NOT work — `postgres` superuser still exists in postgres:17-alpine (the entrypoint creates it implicitly when POSTGRES_USER is non-default), BUT the password is the same `POSTGRES_PASSWORD`. ALTERNATIVELY, run all commands as `-U app` against the `postgres` system database (DROP DATABASE requires being connected to a DIFFERENT db). Use this pattern: connect as `-U app -d postgres` to drop/create the `router` database. App role has CREATEDB only if granted — verify by adding `ALTER USER app CREATEDB;` to Plan 01's `01-init.sql` if missing. Actually CLEANEST: use the official `postgres` superuser role (exists by default per the official image), but the password is the same POSTGRES_PASSWORD. Both `-U postgres` and `-U app` use the same password env var.

    ### Restore drill

    The restore drill is **destructive** — it drops and recreates the `router` database. Only run it when you have a recent dump.

    ```bash
    # Confirms interactively unless --yes is passed.
    bin/restore-drill.sh router-2026-05-14T12.dump
    bin/restore-drill.sh --yes router-2026-05-14T12.dump
    ```

    The script:
    1. Terminates any active connections to the `router` database.
    2. Drops and recreates the database (empty, owned by `app`, with `pgcrypto`).
    3. Runs `pg_restore` from the dump file.
    4. Asserts `SELECT COUNT(*) FROM request_log` returns a numeric value.
    5. Exits 0 on success with a `PASS` line.

    ### Schema evolution
