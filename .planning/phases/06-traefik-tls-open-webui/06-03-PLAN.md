<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tailnet user → chat.<tailnet>.ts.net | Tailscale terminates TLS; forwards HTTP to Traefik on 127.0.0.1:80; Traefik routes by Host header to openwebui:8080 with `webui-basic-auth@docker` middleware gating access. |
| Compose env → OWUI env | All 10 OWUI env vars come from .env via Compose interpolation. OPENAI_API_KEYS contains the router bearer token — never goes to logs (OWUI logs are stdout JSON; bearer is in env, not in request log shape). |
| OWUI → router (app network) | OWUI's OpenAI connector sends `Authorization: Bearer ${ROUTER_BEARER_TOKEN}` to `http://router:3000/v1/...`. Router's bearer-auth middleware (Phase 2) validates. |
| OWUI → postgres (data network) | OWUI uses DATABASE_URL to connect to its dedicated `openwebui` database. The `app` postgres role has access to both `router` and `openwebui` databases (Phase 5 D-B6); database-level isolation prevents schema cross-contamination. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-10 | Elevation of Privilege | OWUI signup-then-admin takeover | mitigate | WEBUI_AUTH=False from boot zero (D-C3 / Pitfall 2) + ENABLE_SIGNUP=False (Pattern 5 belt-and-suspenders). The openwebui DB is empty at first boot per Phase 5 D-B6 — precondition verified. Traefik basic-auth middleware (D-C4) gates the edge as the actual auth surface. |
| T-06-11 | Denial of Service / Information Disclosure | OWUI bypass to backends | mitigate | ENABLE_OLLAMA_API=False (D-C6 / Pitfall 8) + openwebui networks limited to [app, data] — NOT backend. OWUI can ONLY reach the router; the backend network is invisible to it. |
| T-06-12 | Information Disclosure | OWUI bearer token leak via logs | accept | OWUI logs are stdout JSON; bearer lives in the OPENAI_API_KEYS env, not in per-request logs. Single-user single-host posture bounds the threat. Traefik access log disabled (Plan 06-01 T-06-03). |
| T-06-13 | Tampering | OPENAI_API_BASE_URLS rotated to attacker-controlled URL | accept | .env is operator-controlled; same trust boundary as ROUTER_BEARER_TOKEN. Tailscale-only reach (D-A1) bounds the population. |
| T-06-14 | Information Disclosure | WEBUI_SECRET_KEY rotates on container recreate → encrypted-at-rest data unreadable | mitigate | WEBUI_SECRET_KEY pinned from $OWUI_SECRET_KEY in .env (Pitfall 10). Backups round-trip cleanly. |
</threat_model>
