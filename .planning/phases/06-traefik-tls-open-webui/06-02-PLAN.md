<objective>
Surgically mutate the existing `router:` Compose service to make Traefik its only edge. This plan removes the `127.0.0.1:3000:3000` host port binding (the literal contract Phase 5 D-C5 / `router/src/auth/bearer.ts:5-12` TODO depends on for the `/metrics` blackhole at the edge), adds the `edge` network to router's network list (so Traefik can discover and route to it), and adds the six Traefik labels that wire the `router-edge` router to `Host(\`router.${TAILNET_HOSTNAME}.ts.net\`)` with the `metrics-blackhole@file` middleware attached.

This plan deliberately does NOT modify `router-dev:` — per CONTEXT Claude's Discretion and 06-RESEARCH §Pattern 6, the dev-mode router keeps its `127.0.0.1:3000:3000` bind for fast local iteration without Tailscale roundtrip. The `--profile dev` opt-in keeps it out of `docker compose up` default behavior. Document this in Plan 06-04's README mutations.

This plan owns ONLY the router: service block edits in compose.yml. Plan 06-01 owns the traefik: service block; Plan 06-03 owns the openwebui: service block. File-level collision on compose.yml is serialized via wave assignment (Wave 2 after Wave 1).

Purpose:
- EDGE-01: Router becomes addressable at `router.<tailnet>.ts.net` (TLS upstream at Tailscale Serve per D-A2).
- EDGE-02: Router joins the `edge` network — completing its membership in all four networks per Phase 1 D-13 + ROADMAP SC2 "the router is the only service that joins all four".
- EDGE-03: Zero `0.0.0.0:*` or `127.0.0.1:*` mappings on the prod router service.
- EDGE-06: Once Plan 06-04's 120s+ smoke runs, the router-edge chain (Tailscale → Traefik → router → Ollama) supports a long generation without 502 (this plan provides the wiring; the smoke proves it).
- D-B1 (Phase 5 carry-forward): The `metrics-blackhole@file` middleware created in 06-01 is now attached to the router-edge router; external GET /metrics → 404.
- Pitfall 12 mitigation: `traefik.docker.network=${COMPOSE_PROJECT_NAME}_app` label forces Traefik to use the app network (router is now on 4 networks; without the disambiguation label Traefik picks the first one alphabetically, which would be `app` here, but explicit > implicit per the canonical Traefik 2026 pattern).

Output:
- `compose.yml` router: service mutated — ports removed, edge added, 6 Traefik labels added.
- `compose.yml` router-dev: service unchanged (host port preserved per Pattern 6).
- `docker compose config` parses; Traefik discovers the router-edge router on `app` network; internal Docker-DNS scrape of `/metrics` still succeeds.
</objective>
