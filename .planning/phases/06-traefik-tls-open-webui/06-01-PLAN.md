<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Tailscale tailnet → host loopback | Tailscale Serve terminates TLS at the host edge; forwards cleartext HTTP to 127.0.0.1:80 (Traefik). Tailscale ACL gates which tailnet members can reach the services. |
| Traefik container → Docker socket | Traefik reads /var/run/docker.sock READ-ONLY for service discovery — write access would let Traefik launch arbitrary containers. |
| Compose env-var interpolation → Traefik labels | TRAEFIK_BASIC_AUTH value crosses from .env (user-controlled) into a Compose label (rendered into Traefik config). Malformed values can cause middleware misconfiguration but not RCE (Traefik treats label values as opaque strings). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Information Disclosure | Traefik dashboard | mitigate | `api.dashboard: false` + `api.insecure: false` in static config (D-D1). No `traefik.<tailnet>.ts.net` Tailscale service published (D-D2). |
| T-06-02 | Information Disclosure | Docker socket mount | mitigate | Mount as `:ro` (read-only). Traefik cannot launch containers via the read-only socket. |
| T-06-03 | Information Disclosure | Traefik access logs | mitigate | `accessLog: {}` (disabled in v1) — prevents bearer-token leak through default access log format. Phase 9 may revisit with `headers.defaultMode: drop` if logs are needed. |
| T-06-04 | Tampering | Compose label interpolation | accept | TRAEFIK_BASIC_AUTH is operator-supplied via .env; trusted source. Compose's $$→$ collapse is documented behavior. Single-user single-host posture (PROJECT.md) bounds the threat. |
| T-06-05 | Denial of Service | Traefik OOM via large request body | accept | Single-user stack; no attacker-controlled clients. Tailscale-only reach (D-A1) bounds the population. Defer hardening to Phase 9. |
</threat_model>
