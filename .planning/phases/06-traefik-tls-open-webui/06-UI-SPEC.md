# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Traefik + TLS + Open WebUI.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## TL;DR — No Custom UI This Phase

**Phase 6 ships ZERO custom UI code.** There is no React app, no HTML, no CSS, no custom components, no design tokens introduced, no icon library selected, no font chosen. This phase wires up a fully-themed third-party application behind a reverse proxy and a browser-native auth challenge. The "UI" in the phase name comes from "Open WebUI" — a packaged web app consumed verbatim from the upstream container image.

If you are an executor reading this expecting design tokens to apply: **there is no executor-authored UI surface to apply them to.** Every visual decision is owned upstream (Open WebUI maintainers) or by the browser (basic-auth dialog). Phase 6's work is Compose, Traefik static + dynamic config, Tailscale Serve host config, and shell smoke scripts — none of which the user *sees*.

**Visual contract = whatever `ghcr.io/open-webui/open-webui:v0.9.0` ships, with four documented env-var-driven branding hooks.**

- **Custom Open WebUI theme / branding / CSS overrides** — not in v1. Whatever OWUI 0.9.0 ships is what users see.
- **Custom 404 / 502 / 503 error pages from Traefik** — Traefik defaults only.
- **Open WebUI MCP / side-by-side compare UI** — deferred to v2 (per CONTEXT.md `<deferred>`).
- **Grafana dashboards** — Phase 7 (OBS-04). Not this phase.
- **Router admin UI / model registry UI** — never. The project ships a CLI + YAML registry only.

## Branding Hooks (Env-Var Driven)

The only Phase 6 "UI" levers — all owned by Compose env vars, all consumed by the third-party OWUI binary on first boot. No custom code applies the values; OWUI does it internally.

| Hook | Env Var | Value (Phase 6 default) | Source | Effect |
|------|---------|-------------------------|--------|--------|
| App name in chrome | `WEBUI_NAME` | `local-llms` | CONTEXT.md D-C-claude-discretion + `<specifics>` skeleton | Replaces the default "Open WebUI" string in the browser tab title and the OWUI top-left wordmark. Cosmetic only. |
| Auth posture | `WEBUI_AUTH` | `False` | CONTEXT.md D-C3 (irreversible from first boot — Pitfall 10) | Removes OWUI's built-in login/signup screens entirely. Users land directly in the chat surface (after Traefik basic-auth gate clears). |
| Provider connection (sole) | `OPENAI_API_BASE_URLS` | `http://router:3000` (no `/v1` suffix) | CONTEXT.md D-C2 / WEBUI-02 | OWUI's model picker auto-populates from `/v1/models` against this URL. The user sees router models in the OWUI dropdown — visual contract = whatever OWUI renders the list as. |
| Ollama auto-detect | `ENABLE_OLLAMA_API` | `False` | CONTEXT.md D-C6 defense-in-depth | Prevents OWUI from auto-adding a second backend tile / connection. Visible effect: only one provider source in OWUI's connections list (the router). |

**Optional executor-discretion env vars** (not load-bearing, but planner may set for usability):
- `WEBUI_FAVICON` — keep default unless an asset exists in the repo to ship (none does as of Phase 6).
- `DEFAULT_USER_ROLE`, `ENABLE_SIGNUP`, etc. — all moot once `WEBUI_AUTH=False`.
