<!-- BackendAdapter interface widening (router/src/backends/adapter.ts) -->

ADD a new method to the BackendAdapter interface (after probeLiveness on line 62):

  /**
   * OpenAI-shape embedding call. Returns the OpenAI SDK's CreateEmbeddingResponse
   * shape: { object: 'list', data: [{object: 'embedding', index, embedding: number[] | string}], model, usage: {prompt_tokens, total_tokens} }.
   *
   * Adapters whose backend does not support embeddings throw CapabilityNotSupportedError
   * (e.g. LlamacppOpenAIAdapter — llama.cpp-server does not expose /v1/embeddings as of Phase 7).
   * The capability gate in routes/v1/embeddings.ts fires FIRST against entry.capabilities;
   * the adapter-level throw is a defense-in-depth in case capabilities are misdeclared.
   *
   * Adapters: OllamaOpenAIAdapter (Phase 7), VLLMOpenAIAdapter (Phase 7); Phase 8: OllamaCloudAdapter.
   */
  embeddings(
    input: string | string[],
    model: string,
    signal: AbortSignal,
  ): Promise<{
    object: 'list';
    data: Array<{ object: 'embedding'; index: number; embedding: number[] | string }>;
    model: string;
    usage: { prompt_tokens: number; total_tokens: number };
  }>;

The return type matches `OpenAI.Embeddings.CreateEmbeddingResponse` from the openai npm package. Importing the SDK type directly is acceptable IF the type is re-exported; otherwise inline the structural type as above (matches what the SDK actually returns, type-checked by callers).

<!-- Route handler (router/src/routes/v1/embeddings.ts — NEW FILE) -->

Follow chat-completions.ts pattern exactly, simplified for non-streaming:

  - zod schema EmbeddingsRequestSchema:
      model: z.string().min(1)
      input: z.union([z.string().min(1), z.array(z.string().min(1)).min(1)])  // Pitfall E-1 — non-empty enforcement at the route boundary
      encoding_format: z.enum(['float', 'base64']).optional()
      dimensions: z.number().int().positive().optional()
      user: z.string().optional()
      passthrough() at the end (forward-compat with future SDK additions)

  - RegisterEmbeddingsOpts:
      registry: RegistryStore
      makeAdapter: AdapterFactory
      semaphores: { get(backend: string): BackendSemaphore }
      recordOutcome: RecordRequestOutcome

  - Route body (typed.post('/v1/embeddings', {schema: {body: EmbeddingsRequestSchema}}, async (req, reply) => {...})):
      const t0 = performance.now();
      const entry = opts.registry.resolve(body.model);            // throws RegistryUnknownModelError → centralized handler emits 404/openai-envelope
      if (!entry.capabilities.includes('embeddings')) {
        throw new CapabilityNotSupportedError(entry.name, 'embeddings');  // → 400 with openai error envelope
      }
      const adapter = opts.makeAdapter(entry);                    // factory dispatch via Plan 07-03's updated map
      const sem = opts.semaphores.get(entry.backend);
      const release = await sem.acquire();                        // queues or throws BackendSaturatedError → 429
      const ac = new AbortController();
      req.raw.on('close', () => { if (!reply.sent) ac.abort(); });
      try {
        const result = await adapter.embeddings(body.input, entry.backend_model, ac.signal);
        const durationMs = performance.now() - t0;
        opts.recordOutcome({
          protocol: 'openai',
          route: '/v1/embeddings',
          backend: entry.backend,
          model: entry.name,
          statusClass: 'success',
          httpStatus: 200,
          durationMs,
          tokensIn: result.usage?.prompt_tokens ?? 0,
          tokensOut: 0,                                            // per 07-RESEARCH Open Question 3 — emit 0 not NULL
          requestId: req.id,
          timestamp: new Date(),
          agentId: (req as { agentId?: string }).agentId,
        });
        return result;
      } finally {
        release();                                                  // safeRelease idempotency pattern from Phase 3 — semaphore released even on throw
      }

  - Centralized error handling in app.ts already maps CapabilityNotSupportedError + RegistryUnknownModelError + BackendSaturatedError to the right HTTP status + openai-envelope JSON (Phase 4 work). The throw branch's recordOutcome is wired by the same centralized handler (Phase 5 D-C6). No additional error-path recordOutcome needed in the route itself.
