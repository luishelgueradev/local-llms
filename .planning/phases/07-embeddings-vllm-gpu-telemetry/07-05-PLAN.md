  ### vLLM profile commands
  - Default `docker compose up -d` does NOT start vLLM (profile-gated).
  - Start vLLM (chat + embed pair): `docker compose --profile vllm up -d vllm vllm-embed`
  - Switch from Ollama-hot to vLLM-hot: `docker compose stop ollama && docker compose --profile vllm up -d vllm vllm-embed`
  - Cold-start expectation: vLLM's first boot takes up to 10 minutes (JIT compile + CUDA graphs ŌĆö Pitfall V-2). The healthcheck's 600s start_period covers this; watch `docker compose logs -f vllm` for `Capturing CUDA graphs` progress.

  ### Embeddings curl recipes
  - Two embedding-capable models in `models.yaml`: `bge-m3-ollama` (Ollama, always available with default profile) and `bge-m3-vllm` (requires --profile vllm).
  - Pull bge-m3 into Ollama once: `docker compose exec ollama ollama pull bge-m3`
  - Ollama:
    ```
    curl -fsS -X POST http://localhost:3000/v1/embeddings \
      -H "Authorization: Bearer $ROUTER_BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"model":"bge-m3-ollama","input":"Hola mundo"}' | jq '.data[0].embedding | length'
    # Expected: 1024
    ```
  - vLLM-embed (after --profile vllm is up):
    ```
    curl -fsS -X POST http://localhost:3000/v1/embeddings \
      -H "Authorization: Bearer $ROUTER_BEARER_TOKEN" \
      -H "Content-Type: application/json" \
      -d '{"model":"bge-m3-vllm","input":"Hola mundo"}' | jq '.data[0].embedding | length'
    # Expected: 1024 (same dimensions; Pitfall E-2 ŌĆö both backends serve dense 1024-dim BGE-M3)
    ```

  ### Grafana access
  - Subdomain (when Tailscale Service `svc:grafana` is registered): https://grafana.<TAILNET_HOSTNAME>.ts.net
  - LAN bypass (until the operator registers the service in the Tailscale admin console ŌĆö Phase 6 pattern):
    ```

         -u "<TRAEFIK_BASIC_AUTH_USER>:<password>" \

    ```
  - Grafana admin user: `admin` / `$GRAFANA_ADMIN_PASSWORD`. Generate with `openssl rand -hex 24` and store in `.env`.
  - Dashboard URL once logged in: `/d/local-llms/local-llms-router-gpu-backends` (the dashboard's `uid` is `local-llms`).

  ### Known operator steps (one-time, post-deploy)
  - **Pitfall P-2 ŌĆö Prometheus directory permissions:** Prometheus runs as UID 65534. If the host bind-mount at `${HOST_DATA_ROOT}/prometheus/` is root-owned, prom exits with `opening storage failed: permission denied`. Fix once:
    ```
    sudo chown -R 65534:65534 ${HOST_DATA_ROOT}/prometheus
    ```
  - **Pitfall G-3 ŌĆö WSL2 libnvidia-ml.so fallback:** On most WSL2 + Docker Desktop setups, the standard Linux paths (`/usr/lib/x86_64-linux-gnu/libnvidia-ml.so.1`) work via NCT projection. If `docker compose logs nvidia_gpu_exporter` shows `libnvidia-ml.so: cannot open shared object`, uncomment the WSL2 fallback line in compose.yml (next to the existing bind-mounts on the nvidia_gpu_exporter service block; the YAML comment marks the exact line).
  - **Tailscale Service `svc:grafana`:** until the operator runs `tailscale serve --service=svc:grafana` (post-admin-console registration of the service), Grafana is reachable only via the LAN bypass above. Same pattern as Phase 6's `svc:router` + `svc:chat`.
