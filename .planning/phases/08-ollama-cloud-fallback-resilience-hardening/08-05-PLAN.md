<objective>
Enforce CLOUD-04: hard-cap `max_tokens` at 16,384 for cloud-served models. Requests above the cap are REJECTED with a structured error (never silently clipped — D-C1).

## MVP slice user story

After Plan 08-04 closes CLOUD-03 (breaker), this plan closes the OTHER half of the cloud-resilience picture: cost-bounded request shapes. An agent that asks for `max_tokens: 32768` on a cloud model gets a deterministic 400 explaining the cap rather than a surprise weekly-quota burn from a 16k truncation it didn't notice. Local models are unaffected — operators with a local 70B model whose context allows 32k+ tokens are free to use it.

## Scope

- Two routes gated: `/v1/chat/completions` (OpenAI surface) and `/v1/messages` (Anthropic surface — same body field name `max_tokens` per Plan 04's canonical contract).
- Embeddings + count_tokens routes are NOT gated — they don't have a max_tokens parameter.
- Cap is a constant in router/src/config/constants.ts. Not env-configurable in v1 (D-C2). A future Ollama Cloud policy change might bump the cap; flip the constant in a follow-up plan and ship.

Output: 1 new constant + 1 new error class + 2 route edits + 2 test files. Smallest substantive plan in Wave 2; ~15% context.
</objective>

<!-- envelope.ts — new error class -->

After CapabilityNotSupportedError class, add:

  /**
   * Plan 08-05 (CLOUD-04 / D-C1): thrown by the chat / messages routes when
   * a `backend: ollama-cloud` request specifies `max_tokens > CLOUD_MAX_TOKENS_CAP`.
   * Maps to HTTP 400 + structured envelope on both wire surfaces.
   *
   *   - OpenAI envelope:
   *       { error: {
   *           message,
   *           type: 'invalid_request_error',
   *           code: 'cloud_max_tokens_exceeded',
   *           param: 'max_tokens'
   *       } }
   *   - Anthropic envelope:
   *       { type: 'error', error: { type: 'invalid_request_error', message } }
   *
   * The error carries both the requested value and the cap so the client
   * can construct a helpful retry payload (e.g., "use max_tokens=16384 instead").
   */
  export class CloudMaxTokensExceededError extends Error {
    readonly code: 'cloud_max_tokens_exceeded' = 'cloud_max_tokens_exceeded';
    constructor(
      public readonly requested: number,
      public readonly cap: number,
      public readonly modelName: string,
    ) {
      super(
        `Cloud model "${modelName}" rejects max_tokens=${requested}: hard cap is ${cap}. ` +
          `Reduce max_tokens to <= ${cap} and retry; cloud-served models cannot exceed this limit.`,
      );
      this.name = 'CloudMaxTokensExceededError';
    }
  }
