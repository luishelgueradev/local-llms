<objective>
Implement the per-backend circuit breaker per CLOUD-03 + D-B1..D-B4. After 5 failures in a 30s sliding window, the breaker for a given backend trips open for 60s. While open, requests to that backend fail fast with 503 instead of consuming GPU time / cloud quota. Half-open recovery lets ONE probe through after cooldown; success closes, failure re-opens.

## MVP slice user story

After Plans 08-01 (Valkey foundation) and 08-02 (cloud adapter), a real cloud outage (think: Ollama Cloud DDoS, weekly quota exhaustion, DNS failure) would cause every request to hang for 60-120s waiting on the SDK timeout. This plan completes the resilience slice: after 5 such failures within 30s, every subsequent request to the cloud backend fails in <1ms with a structured 503, freeing the agent to fall back or back off. After 60s, ONE probe request goes through — if it succeeds, normal traffic resumes; if not, another 60s of fast-failure follows.

Per-backend scope (D-B4): a breaker on `ollama-cloud` does not affect `ollama` local. An agent that has both can keep serving local requests during a cloud outage. This is the asymmetry the project explicitly wants.

## Why Valkey-backed state

The breaker state is per-process today (single router) but Valkey-backed gives:
- Crash recovery: if the router restarts mid-outage, the breaker state survives the 60s cooldown remains accurate.
- Future multi-instance routers (not in v1 scope but cheap to enable): both instances see the same breaker state, so failures coordinate.
- Auditability: `valkey-cli KEYS 'breaker:*'` shows operator-visible state for debugging.

The downside (round-trip latency on the check) is mitigated by:
- check() is one GET (≤1ms intra-host).
- recordFailure/Success are fire-and-forget (no await blocking the request).

## Out of scope

- Per-(backend, model) granularity (D-B4 rejected): all models on a backend trip together.
- Adaptive thresholds (Hystrix-style sliding-window-based): fixed 5/30s is enough.
- Half-open with N>1 concurrent probes: 1 probe is enough and avoids re-flooding upstream.

Output: 1 new resilience module + 1 new error class + 3 env fields + 4 source edits (app.ts + 3 routes) + 2 test files. Heaviest plan in Wave 2; ~30-35% context.
</objective>

```typescript
// router/src/resilience/circuitBreaker.ts — Phase 8 (CLOUD-03 + D-B1..D-B4).
//
// Per-backend circuit breaker with three states: closed → open → half-open → closed.
//
// State storage (Valkey):
//   breaker:${backend}:fail_count   (INCR, EXPIRE = CIRCUIT_WINDOW_MS, reset on success/state transition)
//   breaker:${backend}:state         ('open' | 'half-open' — closed = key absent; SETEX with CIRCUIT_COOLDOWN_MS)
//   breaker:${backend}:probe_at      (epoch_ms when the next probe is allowed; set on transitioning to open)
//   breaker:${backend}:probe_lock    (SETNX during half-open probe; ensures only ONE probe runs concurrently)
//
// All keys are namespaced by backend so a cloud failure storm does not affect
// the local Ollama keys. The Phase 8 CONTEXT (D-B4) pins per-backend scope.
//
// Concurrency semantics for half-open:
//   - check() observes state and probe_at — if state=open && now >= probe_at,
//     transitions to half-open atomically (Lua eval would be ideal; for the
//     v1 scope we use SET NX on a transition key + a 2-step check that's
//     race-tolerant — duplicate transitions to half-open are harmless because
//     the SECOND probe_lock SETNX fails and returns 'open' to that caller).
//   - The half-open probe acquires probe_lock SETNX with TTL=cooldown. While
//     the lock is held, other check() calls see state=half-open AND
//     probe_lock present → return 'open' (probe in flight).
//   - On probe success: clear all 4 keys (DEL); recordSuccess closes the breaker.
//   - On probe failure: SET probe_at = now + cooldown; SET state = 'open';
//     DEL probe_lock; the breaker re-arms for another full cooldown window.
//
// Failure classification (D-B1):
//   - APIConnectionError (openai SDK — covers ECONNREFUSED, ECONNRESET,
//     ENOTFOUND, fetch network errors): TRIP
//   - APIConnectionTimeoutError (timeout): TRIP
//   - HTTP status >= 500 from upstream: TRIP (the SDK wraps these as
//     APIError with .status — check both Error.status and Error.statusCode)
//   - HTTP 4xx (status < 500 && >= 400): NO TRIP (client error, not backend)
//   - APIUserAbortError: NO TRIP (client gone — not a backend problem)
//   - ZodError / ZodFastifySchemaValidationError: NO TRIP (validation, not backend)
//   - Any other Error: NO TRIP (defensive — unknown errors get treated as
//     client-side bugs; if the breaker should trip on them, operator widens
//     the classifier in a follow-up).

import type { ValkeyClient } from '../clients/valkey.js';
import type { Env } from '../config/env.js';
import type { Logger } from 'pino';
import { APIConnectionError, APIConnectionTimeoutError, APIUserAbortError } from 'openai';
import { z } from 'zod/v4';

export type BreakerState = 'closed' | 'open' | 'half-open';

export interface CircuitBreaker {
  check(backend: string): Promise<{ state: BreakerState }>;
  recordFailure(backend: string, err: unknown): Promise<void>;
  recordSuccess(backend: string): Promise<void>;
  /**
   * For tests: clear all state for a backend. Not exposed at runtime to
   * routes — the natural reset path is recordSuccess on a probe.
   */
  reset(backend: string): Promise<void>;
}

export interface MakeCircuitBreakerOpts {
  valkey: ValkeyClient;
  log: Logger;
  env: Pick<Env, 'CIRCUIT_FAILURE_THRESHOLD' | 'CIRCUIT_WINDOW_MS' | 'CIRCUIT_COOLDOWN_MS'>;
  /** For tests — defaults to Date.now. */
  now?: () => number;
}

const keys = {
  failCount: (b: string) => `breaker:${b}:fail_count`,
  state: (b: string) => `breaker:${b}:state`,
  probeAt: (b: string) => `breaker:${b}:probe_at`,
  probeLock: (b: string) => `breaker:${b}:probe_lock`,
};

/**
 * Classify a thrown error as a breaker trip (true) or not (false).
 * D-B1 — exported for unit testing.
 */
export function isBreakerTrip(err: unknown): boolean {
  if (err instanceof APIUserAbortError) return false;
  if (err instanceof z.ZodError) return false;
  if (err instanceof APIConnectionTimeoutError) return true; // BEFORE APIConnectionError (subclass)
  if (err instanceof APIConnectionError) return true;
  // SDK APIError exposes .status; Node fetch errors expose .code (ENOTFOUND etc.)
  if (typeof err === 'object' && err !== null) {
    const e = err as { status?: number; statusCode?: number; code?: string };
    if (typeof e.status === 'number') {
      return e.status >= 500;
    }
    if (typeof e.statusCode === 'number') {
      return e.statusCode >= 500;
    }
    if (e.code === 'ENOTFOUND' || e.code === 'ECONNREFUSED' || e.code === 'ECONNRESET') {
      return true;
    }
  }
  return false;
}

export function makeCircuitBreaker(opts: MakeCircuitBreakerOpts): CircuitBreaker {
  const { valkey, log, env } = opts;
  const now = opts.now ?? (() => Date.now());

  return {
    async check(backend: string): Promise<{ state: BreakerState }> {
      const [stateRaw, probeAtRaw] = await Promise.all([
        valkey.get(keys.state(backend)),
        valkey.get(keys.probeAt(backend)),
      ]);
      if (!stateRaw) return { state: 'closed' };
      const probeAt = probeAtRaw ? Number(probeAtRaw) : 0;
      const t = now();
      if (stateRaw === 'open') {
        if (t >= probeAt) {
          // Transition to half-open: try to acquire the probe lock.
          // SETNX returns 1 if acquired, 0 if already held.
          const acquired = await valkey.set(
            keys.probeLock(backend),
            String(t),
            'PX',
            env.CIRCUIT_COOLDOWN_MS,
            'NX',
          );
          if (acquired === 'OK') {
            await valkey.set(keys.state(backend), 'half-open', 'PX', env.CIRCUIT_COOLDOWN_MS);
            log.info({ backend }, 'circuit breaker transitioning to half-open (probe acquired)');
            return { state: 'half-open' };
          }
          // Lock held by another concurrent call → still open from this caller's POV.
          return { state: 'open' };
        }
        return { state: 'open' };
      }
      if (stateRaw === 'half-open') {
        // Another probe is in flight (probe_lock should exist). Treat as open for this caller.
        return { state: 'open' };
      }
      // Unknown state value (shouldn't happen); fail safe → closed.
      log.warn({ backend, stateRaw }, 'circuit breaker: unknown state value, treating as closed');
      return { state: 'closed' };
    },

    async recordFailure(backend: string, err: unknown): Promise<void> {
      if (!isBreakerTrip(err)) return;

      const stateRaw = await valkey.get(keys.state(backend));
      if (stateRaw === 'half-open') {
        // Probe failed — re-open for another cooldown.
        const t = now();
        await Promise.all([
          valkey.set(keys.state(backend), 'open', 'PX', env.CIRCUIT_COOLDOWN_MS),
          valkey.set(keys.probeAt(backend), String(t + env.CIRCUIT_COOLDOWN_MS), 'PX', env.CIRCUIT_COOLDOWN_MS),
          valkey.del(keys.probeLock(backend)),
        ]);
        log.warn({ backend }, 'circuit breaker: probe failed, re-opening');
        return;
      }

      // closed (or stale) — increment fail counter.
      const count = await valkey.incr(keys.failCount(backend));
      if (count === 1) {
        // First failure in this window — set TTL.
        await valkey.pexpire(keys.failCount(backend), env.CIRCUIT_WINDOW_MS);
      }
      if (count >= env.CIRCUIT_FAILURE_THRESHOLD) {
        const t = now();
        await Promise.all([
          valkey.set(keys.state(backend), 'open', 'PX', env.CIRCUIT_COOLDOWN_MS),
          valkey.set(keys.probeAt(backend), String(t + env.CIRCUIT_COOLDOWN_MS), 'PX', env.CIRCUIT_COOLDOWN_MS),
        ]);
        log.warn({ backend, threshold: env.CIRCUIT_FAILURE_THRESHOLD, windowMs: env.CIRCUIT_WINDOW_MS }, 'circuit breaker: opening');
      }
    },

    async recordSuccess(backend: string): Promise<void> {
      const stateRaw = await valkey.get(keys.state(backend));
      if (stateRaw === 'half-open') {
        // Probe succeeded — close the breaker.
        await Promise.all([
          valkey.del(keys.state(backend)),
          valkey.del(keys.failCount(backend)),
          valkey.del(keys.probeAt(backend)),
          valkey.del(keys.probeLock(backend)),
        ]);
        log.info({ backend }, 'circuit breaker: probe succeeded, closing');
        return;
      }
      // closed → no-op (we don't reset the counter on every success; it
      // expires naturally via TTL). This is the sliding-window approximation
      // — a fixed window with TTL is enough for the threshold to be
      // meaningful without Lua sliding-window math.
    },

    async reset(backend: string): Promise<void> {
      await Promise.all([
        valkey.del(keys.state(backend)),
        valkey.del(keys.failCount(backend)),
        valkey.del(keys.probeAt(backend)),
        valkey.del(keys.probeLock(backend)),
      ]);
    },
  };
}
```

The implementation deliberately uses fixed-window counters (`fail_count` with TTL) rather than a Lua-scripted sliding-window — the approximation is sufficient at the threshold/window/cooldown scales we care about (5/30s/60s). A Lua scaled sliding window can be added in a future plan if observed false negatives.
