<objective>
Add a 30-second Valkey-backed read-through cache for the parsed models.yaml registry per D-D4 + DATA-06 (the second Valkey consumer in 08-CONTEXT's enumerated list).

## MVP slice user story

In v1 single-router this is a performance optimization (warm cache on restart avoids YAML re-parse + zod re-validation). The bigger payoff is structural: when v2 brings multi-instance routers (sticky-session via Traefik OR pure load balancing), each instance reads the same cached Registry — operator updates models.yaml on one instance, the file watcher invalidates the cache, the other instances see the new shape on next refresh.

For v1, the user-visible benefit is "router boots a tick faster after restart"; the architectural benefit is "registry now coordinates via Valkey, so future multi-instance is structurally ready."

## Why CACHE the registry rather than relying on file reads

- Multi-instance readiness: the cache is the shared state.
- Reload propagation: index.ts's fs.watch listens to the LOCAL file; in multi-instance, fs.watch only sees the local node's mounted volume. Valkey-cached state is shared across nodes.
- File-read amortization: re-parsing models.yaml + zod validation is a few ms; cached path is sub-ms.

## Out of scope

- Cache invalidation on remote node's file change (requires PubSub-based invalidation — v2 territory). Current implementation lets the 30s TTL provide the upper-bound staleness.
- Encrypted cache values (Valkey is internal-only on `data` network; cache contents = models.yaml which is in the repo, not secret).

```typescript
// router/src/config/registryCache.ts — Plan 08-09 (DATA-06).
//
// 30s Valkey-backed read-through cache for the parsed models.yaml.
// File is the source of truth; cache is a derivative (D-D4).
import type { Logger } from 'pino';
import type { ValkeyClient } from '../clients/valkey.js';
import { RegistrySchema, type Registry } from './registry.js';

// Cache key includes a version int so a future schema-breaking change can
// invalidate ALL cached blobs by bumping. Not env-configurable in v1.
const CACHE_KEY = 'registry:models-yaml:cache:v1';
const TTL_SEC = 30;

export interface RegistryCache {
  get(): Promise<Registry | null>;
  set(reg: Registry): Promise<void>;
  clear(): Promise<void>;
}

export interface MakeRegistryCacheOpts {
  valkey: ValkeyClient;
  log: Logger;
}

export function makeRegistryCache(opts: MakeRegistryCacheOpts): RegistryCache {
  const { valkey, log } = opts;
  return {
    async get(): Promise<Registry | null> {
      try {
        const raw = await valkey.get(CACHE_KEY);
        if (!raw) return null;
        let parsed: unknown;
        try {
          parsed = JSON.parse(raw);
        } catch (err) {
          log.warn({ err }, 'registry cache: malformed JSON in Valkey; returning null');
          return null;
        }
        // Re-validate through RegistrySchema — defense in depth.
        const result = RegistrySchema.safeParse(parsed);
        if (!result.success) {
          log.warn({ err: result.error.issues[0] }, 'registry cache: schema mismatch; returning null');
          return null;
        }
        return result.data;
      } catch (err) {
        log.warn({ err: err instanceof Error ? err.message : err }, 'registry cache: get failed, returning null');
        return null;
      }
    },
    async set(reg: Registry): Promise<void> {
      try {
        await valkey.set(CACHE_KEY, JSON.stringify(reg), 'EX', TTL_SEC);
      } catch (err) {
        log.warn({ err: err instanceof Error ? err.message : err }, 'registry cache: set failed (non-fatal)');
      }
    },
    async clear(): Promise<void> {
      try {
        await valkey.del(CACHE_KEY);
      } catch (err) {
        log.warn({ err: err instanceof Error ? err.message : err }, 'registry cache: clear failed (non-fatal)');
      }
    },
  };
}
```
