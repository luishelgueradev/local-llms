<objective>
Implement server-side per-token-per-minute rate limit per ROUTE-11 + D-D2/D-D3. Each authenticated request increments a per-bearer-hash counter in Valkey; when the count exceeds `ROUTER_RATE_LIMIT_RPM` (default 600), subsequent requests in that minute receive 429 + `Retry-After: 60`.

## MVP slice user story

An agent SDK with default retry-on-failure and a buggy loop can fire hundreds of requests per minute against a slow local backend. Without rate limiting, this would saturate the GPU and queue (Phase 3 semaphore) but also rack up unbounded Postgres rows + log spam. With this plan, request #601 returns 429 instantly; the agent's exponential-backoff (or the operator's manual fix) takes over. The local GPU is protected; the operator's bandwidth is protected; the Postgres + Valkey writes are bounded.

## Why "per bearer hash" rather than per-IP

D-D2 explicitly: the bearer token IS the identity. Per-IP would lump multiple agents on the same host together; per-token disambiguates if the operator ever generates multiple tokens (not in v1 — single bearer — but the key shape is future-compatible).

The bearer is SHA-256-hashed and truncated to 8 hex chars for the Valkey key — even an attacker with Valkey-CLI access (D-D2 acknowledges the MONITOR risk) can't reverse the hash to the token.

## Why fail-open on Valkey errors

If Valkey crashes, the alternative ("fail closed") would 503 every request — which is strictly worse than no rate limit (legitimate traffic is killed alongside the bad). Plan 08-01's ioredis config (enableOfflineQueue: false + maxRetriesPerRequest: 1 + connectTimeout: 2_000) ensures the hook's INCR throws fast on Valkey-down, and the hook logs+continues. The operator sees `valkey-error` logs + the breaker (Plan 08-04) which trips on backend failures (not Valkey failures), and can intervene.

## Out of scope

- Per-token RPM in models.yaml (D-D3 rejected): global default is fine for single-user.
- Token-bucket / leaky-bucket math: fixed-window per-minute is cheaper and adequate.
- Distributed coordination across multiple router instances: single-router v1; Valkey serves as the coordination point already.

Output: 1 new middleware file + 1 new env field + 1 new error class + 1 app.ts hook addition + 2 test files. ~20% context.
</objective>

```typescript
// router/src/middleware/rateLimit.ts — Phase 8 (ROUTE-11 / D-D2 / D-D3).
//
// Per-bearer-token-per-minute fixed-window rate limit, Valkey-backed.
//
// Key shape: ratelimit:${bearer_hash_8char}:${epoch_minute}
//   - bearer_hash: first 8 hex chars of SHA-256(authorization.slice(7))
//     (after stripping the 'Bearer ' prefix). Hashing protects the token
//     from MONITOR / loglevel verbose observability (D-D2).
//   - epoch_minute: Math.floor(Date.now() / 60_000). Fixed per-minute bucket.
//
// INCR returns the new count. If count === 1 (first request of the minute),
// EXPIRE the key with TTL 60s + a small margin (65s to handle the boundary
// race where the request arrives at second :59.9 of minute N and the EXPIRE
// fires at :00.1 of minute N+1). If count > limit → throw RateLimitExceededError.
//
// Public paths: skipped entirely (same set as auth/bearer.ts). No INCR; no log.
//
// Fail-open: if Valkey throws (network error, auth failure, OOM), log at
// warn and PROCEED (the request goes through). This is the v1 D-D3 default;
// a future plan can swap to fail-closed if observed traffic patterns prefer.
import { createHash } from 'node:crypto';
import type { FastifyReply, FastifyRequest } from 'fastify';
import type { Logger } from 'pino';
import type { ValkeyClient } from '../clients/valkey.js';
import { RateLimitExceededError } from '../errors/envelope.js';
import { PUBLIC_PATHS } from '../auth/bearer.js';

export interface MakeRateLimitOpts {
  valkey: ValkeyClient;
  log: Logger;
  rpmLimit: number;
  /** For tests — defaults to Date.now. */
  now?: () => number;
}

/**
 * Hash a bearer token to 8 hex chars (deterministic, non-reversible).
 * Exported for unit testing.
 */
export function bearerHash(rawBearer: string): string {
  return createHash('sha256').update(rawBearer, 'utf8').digest('hex').slice(0, 8);
}

export function makeRateLimitPreHandler(opts: MakeRateLimitOpts) {
  const { valkey, log, rpmLimit } = opts;
  const now = opts.now ?? (() => Date.now());

  return async function rateLimit(req: FastifyRequest, _reply: FastifyReply): Promise<void> {
    const path = req.url.split('?')[0];
    if (path && PUBLIC_PATHS.has(path)) return;

    const auth = req.headers.authorization;
    if (typeof auth !== 'string' || auth.length < 8) {
      // Bearer auth hook (runs BEFORE this one) should have already thrown
      // BearerAuthError — if we got here without an auth header, something is
      // wrong but it's not the rate-limit hook's job to fix. Skip.
      return;
    }

    // Strip 'Bearer ' prefix (7 chars, case-insensitive — but bearer.ts already
    // validated the scheme so the slice is safe).
    const SCHEME = 'bearer ';
    const supplied = auth.slice(SCHEME.length);
    const hash = bearerHash(supplied);
    const minute = Math.floor(now() / 60_000);
    const key = `ratelimit:${hash}:${minute}`;

    try {
      const count = await valkey.incr(key);
      if (count === 1) {
        // First request of this minute — set TTL with a small margin to
        // tolerate the second-boundary race.
        await valkey.expire(key, 65);
      }
      if (count > rpmLimit) {
        throw new RateLimitExceededError(hash, count, rpmLimit);
      }
    } catch (err) {
      if (err instanceof RateLimitExceededError) throw err;
      // Valkey-down or other infrastructure error — fail open.
      log.warn(
        { err: err instanceof Error ? err.message : String(err), hash, minute },
        'rate-limit: valkey error, failing open',
      );
    }
  };
}
```
